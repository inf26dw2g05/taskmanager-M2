# Capítulo 1 — Introducción

## 1.1. Descripción del proyecto

**TaskManager API** es una aplicación REST desarrollada en el marco de la asignatura *Desarrollo Web II* (M1) del grado en Informática de la UMAIA, curso 2025/2026.

El sistema permite a los usuarios gestionar **proyectos personales** y las **tareas** asociadas a esos proyectos. Cada usuario dispone de un espacio aislado: únicamente puede acceder, crear, modificar y eliminar sus propios proyectos y tareas. Las credenciales y la autorización están gestionadas mediante tokens JWT.

La aplicación está pensada para funcionar como capa de servicios (backend) consumible por cualquier cliente web o móvil. En esta entrega se entrega documentada y verificada mediante una colección de Postman, pero el diseño es independiente del cliente.

## 1.2. Objetivos

El proyecto cumple con los requisitos obligatorios del enunciado:

- Arquitectura **REST**, utilizando los cuatro verbos HTTP (GET, POST, PUT, DELETE).
- **Tres recursos** distintos: `users`, `projects` y `tasks`.
- **Dos relaciones 1:n** entre los recursos: `users → projects` y `projects → tasks`.
- Representaciones en **JSON**.
- **Autenticación y autorización** mediante JWT, con control de acceso por propietario.
- **Logging** del usuario autenticado en consola para cada petición protegida.
- Documentación **OpenAPI 3.0** servida vía Swagger UI.
- **Colección Postman** para consulta de los recursos.
- Despliegue **multi-contenedor** con Docker Compose (MySQL + Node.js).
- **Base de datos MySQL** poblada con al menos 30 registros por tabla.

Adicionalmente, se han incorporado los siguientes elementos valorados en el enunciado:

- Uso del framework **Express**.
- **Aislamiento por propietario** aplicado en `projects` y `tasks` (un usuario solo accede a sus recursos).
- En el capítulo 4 se realiza una **comparación entre la implementación actual y los distintos flows de OAuth 2.0**.
